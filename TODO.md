- Write a script for finding how many tokens are in a replay to help gauge how large the context window will need to be. Ideally, the context window will be able to handle all tokens from a replay, for each player, x~1.5. The input should be able to handle all self tokens, and 0% omission enemy tokens, and then the output would need to be able to handle all of the enemy tokens (self tokens are not drawn on the output canvas). All tokens means every entity and upgrade that exist at a given timestep, added across every timestep for the match. Current estimate is that 4k input and 4k output context should be sufficient. But this should be tested. And the method for truncating both input and output when too large should be validated if existing and implemented and then validated if not existing in the codebase.
